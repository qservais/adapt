## 🎯 MISSION

Tu es un développeur full-stack senior. Tu vas construire **ADAPT by LMJ**, une application de coaching sportif adaptatif. Le cœur du produit est l’**ADAPT ENGINE** : un algorithme qui calcule chaque matin un score composite (0–100) à partir d’un check-in de l’athlète, puis adapte automatiquement sa séance du jour (charge, volume, intensité).

L’application a **deux interfaces** :

- **App mobile-first** pour l’athlète (React responsive, expérience mobile)
- **Dashboard web** pour le coach

## 🧠 ADAPT ENGINE — ALGORITHME COMPLET

Crée `/server/services/adapt-engine.js` avec cette logique EXACTE :

```javascript
/**
 * ADAPT ENGINE — Moteur de décision central
 * Calcule le score adaptatif (0–100) et détermine le mode de séance
 * Ce fichier est le cœur du produit — ne pas modifier sans tests
 */

function calculateAdaptScore({ sleep, energy, stress, soreness, motivation, rpe_yesterday = null, cycle_phase = null }) {
  
  // ÉTAPE 1 — Normalisation (0.0 à 1.0)
  const sleep_n      = (sleep - 1) / 4;
  const energy_n     = (energy - 1) / 4;
  const stress_n     = (stress - 1) / 4;      // inversé : stress élevé = mauvais
  const soreness_n   = (soreness - 1) / 4;    // inversé : courbatures élevées = mauvais
  const motivation_n = (motivation - 1) / 4;

  // ÉTAPE 2 — Score de base pondéré
  let score_base = 
    (sleep_n      * 0.25) +
    (energy_n     * 0.20) +
    ((1 - stress_n)   * 0.15) +
    ((1 - soreness_n) * 0.20) +
    (motivation_n * 0.20);
  // score_base entre 0.0 et 1.0

  // ÉTAPE 3 — Modificateur RPE de la veille
  if (rpe_yesterday !== null) {
    if (rpe_yesterday >= 9)      score_base *= 0.85;  // séance très dure hier
    else if (rpe_yesterday >= 8) score_base *= 0.90;  // séance dure hier
    else if (rpe_yesterday <= 4) score_base *= 1.05;  // séance facile hier → bonus
  }

  // ÉTAPE 4 — Modificateur cycle menstruel (si activé)
  if (cycle_phase) {
    const cycle_modifiers = {
      menstrual:   0.85,  // fatigue accrue
      follicular:  1.00,  // neutre
      ovulatory:   1.05,  // pic d'énergie
      luteal:      0.95,  // légère fatigue
    };
    score_base *= (cycle_modifiers[cycle_phase] || 1.00);
  }

  // ÉTAPE 5 — Score final (0–100, arrondi)
  const adapt_score = Math.round(Math.min(100, Math.max(0, score_base * 100)));

  // ÉTAPE 6 — Détermination du mode
  let session_mode;
  if      (adapt_score >= 80) session_mode = 'performance';
  else if (adapt_score >= 60) session_mode = 'normal';
  else if (adapt_score >= 40) session_mode = 'adapt';
  else                         session_mode = 'recovery';

  return { adapt_score, session_mode };
}

/**
 * Calcule la charge adaptée selon le mode
 * @param {number} nominal_kg - Charge nominale du programme
 * @param {string} mode - Mode ADAPT
 * @returns {number} Charge suggérée en kg
 */
function calculateAdaptedLoad(nominal_kg, mode) {
  if (!nominal_kg) return null;
  const ratios = {
    performance: 1.025,   // +2.5% (milieu de la plage 100-105%)
    normal:      1.00,
    adapt:       0.775,   // milieu de la plage 75-80%
    recovery:    0.20,    // 0-40%, on prend 20% ou poids de corps
  };
  const ratio = ratios[mode] || 1.0;
  return Math.round(nominal_kg * ratio * 4) / 4; // arrondi au 0.25kg
}

module.exports = { calculateAdaptScore, calculateAdaptedLoad };
```

## 🔐 AUTHENTIFICATION

### Stratégie

- **Email/password** : hash bcrypt (rounds: 12), JWT access (1h) + refresh (30 jours)
- **Google OAuth 2.0** : via passport.js + passport-google-oauth20
- **Apple Sign-In** : via passport-apple ou apple-signin-auth
- **Middleware auth** : vérifier le JWT sur chaque route protégée, retourner 401 si expiré
- **RBAC** : middleware `requireRole('coach')` pour les routes /coach/*

### Format JWT payload

```json
{ "userId": "uuid", "role": "athlete|coach", "email": "..." }
```

### Codes d’erreur API standards

```json
{ "error": { "code": "AUTH_TOKEN_EXPIRED", "message": "..." } }
```

Codes métier : `AUTH_INVALID_CREDENTIALS`, `AUTH_TOKEN_EXPIRED`, `AUTH_FORBIDDEN`, `CHECKIN_ALREADY_EXISTS`, `CHECKIN_WINDOW_CLOSED`, `SESSION_NOT_UNLOCKED`, `PAIN_OVERRIDE_ACTIVE`, `COACH_CLIENT_LIMIT`, `INVITE_CODE_INVALID`

## ✅ CRITÈRES D’ACCEPTANCE — CE QUI DOIT FONCTIONNER

### ADAPT ENGINE

- Score max (sommeil=5, énergie=5, stress=1, courbatures=1, motivation=5) → score >= 85, mode “performance”
- Score min (tout à 1 sauf stress et courbatures à 5) → score <= 15, mode “recovery”
- `has_pain = true` → mode forcé “recovery” + alerte P1 créée, quelle que soit le score
- RPE veille = 9 + score base 75 → score final ~64 (mode “normal”)
- Phase menstruelle + score base 70 → score final ~60

### Check-in

- Impossible de soumettre 2 check-ins le même jour (erreur 409)
- Check-in après 14h → erreur 422 `CHECKIN_WINDOW_CLOSED`
- Sans check-in → séance inaccessible (erreur 403 `SESSION_NOT_UNLOCKED`)

### Séances

- Mode ADAPT (score ~45) + charge nominale 100kg → charge affichée 75-80kg
- Mode RECOVERY → aucun exercice avec charge > 40% du nominal
- Coach modifie séance → athlète voit version modifiée avec mention “Mis à jour par ton coach”

### Alertes

- Douleur signalée → alerte P1 dans dashboard coach en < 5 secondes
- Validation P1 par coach → séance déverrouillée pour l’athlète
- 3 jours sans activité → alerte P2 générée au job de 09h00

### Sécurité

- Token athlète sur `/api/coach/clients` → 403 `AUTH_FORBIDDEN`
- Athlète sur profil d’un autre athlète → 403
- Coach sur données d’un athlète non lié → 403
