## PHASE 1 — BUGS À CORRIGER EN PRIORITÉ

### BUG-V3-01 : Séance du jour même toujours pas reçue côté athlète (RÉCURRENT)
Ce bug a déjà été signalé dans le patch V2 (BUG-V2-01) et N'EST TOUJOURS PAS CORRIGÉ.
Quand le coach programme une séance pour le jour même (ex : on est mercredi, le coach envoie une séance pour mercredi), l'athlète ne la voit pas.

Checklist de debug obligatoire — vérifie CHAQUE point :

1. **Timezone** : compare la date stockée en base avec la date côté client. Si le serveur stocke en UTC et que le client est en Europe/Brussels (UTC+1 ou UTC+2 en été), une séance créée pour "mercredi 25 mars" pourrait être stockée comme "mardi 24 mars 23h00 UTC" ou "mercredi 25 mars 01h00 UTC". Normalise les dates : stocke les dates de séance comme DATE PURE (YYYY-MM-DD) sans heure, pas comme datetime.
2. **Filtre de requête côté athlète** : vérifie la query qui fetch les séances du jour. Le filtre doit être `WHERE session_date = '2026-03-25'` (comparaison de date pure), PAS `WHERE session_date >= '2026-03-25T00:00:00Z' AND session_date < '2026-03-26T00:00:00Z'` (qui peut rater selon le timezone).
3. **Cache / state stale** : quand le coach crée la séance, l'app athlète la voit-elle après un pull-to-refresh ? Si oui, c'est un problème de refresh automatique — ajouter un refresh des séances quand l'athlète ouvre l'onglet Séance ou revient sur l'Accueil.
4. **Statut de la séance** : vérifie que la séance créée a bien un statut "active" ou "scheduled" et que le filtre côté athlète n'exclut pas les séances créées le jour même.
5. **Test de bout en bout** : après correction, teste ce scénario exact : le coach crée une séance pour aujourd'hui à 14h → l'athlète ouvre son app à 14h05 → la séance DOIT apparaître sur l'accueil et dans l'onglet Séance.

CE BUG EST BLOQUANT. Ne passe pas à la suite tant qu'il n'est pas corrigé et testé.

### DASH-V3-04 : Champ tempo pour les exercices
Permettre au coach de définir un tempo pour chaque exercice dans une séance :
- Format : 4 chiffres séparés par des tirets (ex : 3-1-2-0)

  - 2e chiffre = pause en position basse en secondes
  - 3e chiffre = phase concentrique (montée) en secondes
  - 4e chiffre = pause en position haute en secondes
- Champ optionnel (pas tous les exercices ont un tempo)
- Input : 4 champs numériques côte à côte avec séparateur "-" ou un seul champ texte avec validation regex `^\d-\d-\d-\d$`
- Côté athlète pendant la séance : afficher le tempo sous les paramètres de l'exercice (ex : "Tempo : 3-1-2-0")
- Optionnel (bonus) : pendant l'exécution, afficher un métronome visuel qui guide le tempo (compteur qui défile pour chaque phase)

## PHASE 3 — APP ATHLÈTE — NUTRITION

### NUTRI-V3-01 : Scan code-barres pour les macros
Dans l'onglet Nutrition, ajouter un bouton "Scanner un produit" :

**Fonctionnement :**
1. L'athlète clique sur "Scanner" → ouvre la caméra du téléphone
2. Scan du code-barres (EAN/UPC) du produit alimentaire
3. Requête vers l'API Open Food Facts (https://world.openfoodfacts.org/api/v2/product/{barcode}.json) — API gratuite, pas de clé nécessaire
4. Récupérer les données nutritionnelles : nom du produit, calories, protéines, glucides, lipides (pour 100g)
5. Afficher les résultats à l'athlète avec un champ "Quantité (g)" pour ajuster les macros proportionnellement
6. Bouton "Ajouter au repas" qui ajoute le produit au repas en cours

**Implémentation technique :**
- Utiliser une librairie de scan code-barres :
  - Web/PWA : `quagga2` ou `html5-qrcode` (utilise la caméra du navigateur)
  - React Native : `react-native-camera` + `react-native-barcode-mask` ou `expo-barcode-scanner`
- Appel API : `GET https://world.openfoodfacts.org/api/v2/product/{barcode}.json`
- Champs à extraire de la réponse : `product.product_name`, `product.nutriments.energy-kcal_100g`, `product.nutriments.proteins_100g`, `product.nutriments.carbohydrates_100g`, `product.nutriments.fat_100g`
- Si le produit n'est pas trouvé : afficher "Produit non trouvé — saisie manuelle" avec les champs habituels
- Historique : sauvegarder les produits scannés récemment pour un accès rapide ("Produits récents")

**UX :**
- Bouton scan bien visible dans l'écran d'ajout de repas (icône code-barres)
- Après le scan : afficher une fiche produit avec les macros, un champ quantité, et le bouton d'ajout
- Le calcul des macros se met à jour en temps réel quand l'athlète change la quantité
